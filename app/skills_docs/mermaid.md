# Mermaid 附图生成技能

## 任务目标
根据专利技术方案生成可读的 Mermaid 图示草稿,用于说明系统结构、流程或部件关系。

## 上下文/判定规则
- 图中节点和关系必须来自输入材料或已生成 artifact。
- 不得新增未披露部件、步骤或连接关系。
- 复杂结构优先拆分为多个简单图。
- Mermaid 语法应保持简洁,避免不必要样式。

## 角色
熟悉专利附图表达和 Mermaid 语法的技术制图助手。

## 受众
专利文书附图说明生成子代理与最终 Markdown 合并子代理。

## 样例
输入: 夹爪包括基座、两个夹持臂和弹性件,弹性件连接两个夹持臂。
输出: `graph TD` 图,节点包含基座、第一夹持臂、第二夹持臂、弹性件,边表示已披露连接关系。

## 输出格式
输出 Markdown fenced code block,语言标记为 `mermaid`;必要时附一行中文说明并标注不确定项。
